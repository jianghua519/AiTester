## 🚀 快速开始

### 第一步: 理解项目

1. 阅读 `necessary_features.md` 了解项目的功能范围和目标
2. 阅读 `architecture_design.md` 了解技术架构和实现方案
3. 查看各个架构图,建立系统的整体认知

### 第二步: 准备开发

1. 阅读 `ai_developer_prompt_template.md`,理解开发规范和约束
2. 阅读 `hyper-detailed_atomic_task_list.md`,了解完整的任务列表

### 第三步: 开始开发

1. 从 `hyper-detailed_atomic_task_list.md` 中选择一个任务(建议从阶段一的第一个任务开始)
2. 使用 `ai_developer_prompt_template.md` 中的提示词模板
3. 将任务信息填入提示词模板
4. 发送给 AI 开发者执行

## 📋 开发流程

每个任务的执行流程:

```
1. 思考 → 理解任务需求
2. 设计 → 更新 DESIGN_DOC.md
3. 编码 → 分步骤生成代码(每次不超过 500 行)
4. 测试 → 编写并执行测试,保存证据
5. 文档 → 生成 TEST_REPORT.md
6. 提交 → 执行 Git commit
```

## ⚠️ 重要约束

- ✅ 代码单次生成不超过 500 行
- ✅ 必须支持中文、日文、英文三语言界面
- ✅ 必须先更新设计文档再编码
- ✅ 必须编写测试用例和生成测试报告
- ✅ Python 必须运行在 venv 环境
- ✅ 所有依赖必须锁定版本号
- ✅ 每完成一个任务必须 Git commit
- ✅ 代码必须符合业界规范(PEP 8 / Prettier)

## 🎯 开发目标

构建一个功能完整、企业级、AI 驱动的自动化测试平台,支持:
- Web、API、移动端自动化测试
- 测试资产管理和团队协作
- AI 赋能(NLP 生成测试、智能诊断、元素自愈)
- 企业级特性(RBAC、审计日志、分布式执行)
